# wd-exec mode in wiredesk-term

## Overview

Add a non-interactive `--exec` mode to the existing `wiredesk-term` binary so AI agents (and shell scripts) can run a single command on the remote host shell via serial and get clean stdout + propagated exit code back. Optional `--ssh ALIAS` flag chains an `ssh -tt` hop on the host, with prompt-detection at both layers (host PowerShell, remote bash) so MOTD / banner / prompts are stripped from the returned stdout.

**Why this approach (not a separate `wd-exec` binary):** reuses `handshake` + transport split + `heartbeat_thread` already in `wiredesk-term`, ships as a single artifact, keeps the same mutual-exclusion model on the serial port. Diff is roughly +200 lines of code + tests in `apps/wiredesk-term/src/main.rs`, no protocol changes, no host changes.

**Brief:** `docs/briefs/wd-exec.md` (signed off by stakeholder, green light).

## Solution Overview

A new `run_oneshot` branch in `run()` activates when `--exec <CMD>` is supplied:

1. Parse `--exec`, `--ssh`, `--timeout` flags via `clap`.
2. Open serial, `try_clone` into reader/writer halves (same as today).
3. Run `handshake()` (existing).
4. Send `ShellOpen { shell: "" }` (default = PowerShell on Windows host).
5. **Do NOT** call `enable_raw_mode` and **do NOT** open stdin — this run is non-interactive.
6. Spawn the existing `heartbeat_thread` so long commands don't trip the host's idle timeout.
7. Read `ShellOutput` packets in a loop, pushing bytes into an in-memory buffer.
8. Walk the buffer line-by-line, looking for the host PowerShell prompt (`PS X:\…> `).
9. (If `--ssh ALIAS`) send `ssh -tt ALIAS\r` as `ShellInput`, then walk for the remote bash prompt (Starship `➜ ` / generic `\$\s*$` / `#\s*$`).
10. Generate a v4 UUID. Format the command:
    - PS-only: `try { <cmd> } catch { $LASTEXITCODE = 1 }; "__WD_DONE_<uuid>__$LASTEXITCODE"\r`
    - bash post-ssh: `<cmd>; echo "__WD_DONE_<uuid>__$?"\r`
11. Send the formatted line as `ShellInput`.
12. Continue accumulating bytes; scan each completed line for the sentinel regex `^__WD_DONE_<uuid>__(\d+)\s*$` (anchored — defeats stdin-echo of the format-string).
13. On match: `exit_code = capture(1)`. Run `clean_stdout` to strip everything before the prompt, the echoed command line (only present in `--ssh` path; absent in PS-only because pipe-mode PS without PSReadLine doesn't echo stdin), and the sentinel line itself.
14. Print clean stdout, send `ShellClose` + `Disconnect` (and possibly a phantom `exit\r` for `--ssh` cleanup), `exit(exit_code)`.
15. Timeout: if no sentinel within `--timeout` seconds, print a diagnostic to stderr and exit 124.

## Technical Details

### New CLI flags

```rust
struct Args {
    // existing: port, baud, shell, name
    /// Run COMMAND non-interactively on the host shell, then exit.
    #[arg(long, value_name = "COMMAND")]
    exec: Option<String>,

    /// When --exec is set: ssh -tt to ALIAS on the host shell first, run COMMAND there.
    #[arg(long, value_name = "ALIAS")]
    ssh: Option<String>,

    /// Seconds to wait for the sentinel before giving up.
    #[arg(long, default_value = "30")]
    timeout: u64,
}
```

### Helpers (pure, unit-testable)

```rust
/// PowerShell prompt: "PS X:\path> " at end of line.
fn is_powershell_prompt(line: &str) -> bool;

/// Remote bash/Starship prompt: "$ ", "# ", "➜ " at end.
fn is_remote_prompt(line: &str) -> bool;

/// Format `cmd` with sentinel for the chosen shell flavour.
/// PS variant wraps in try/catch so terminating errors still emit the sentinel
/// (otherwise the run hangs to timeout 124 instead of returning exit 1).
fn format_command(uuid: &Uuid, kind: ShellKind, cmd: &str) -> String;

/// Look for `__WD_DONE_<uuid>__(\d+)` exactly. Stdin echo with literal
/// `$LASTEXITCODE` / `$?` won't match — `\d+` is digit-only.
fn parse_sentinel(line: &str, uuid: &Uuid) -> Option<i32>;

/// Slice the accumulated buffer: drop everything up to and including the
/// last prompt line; drop the echoed command line if present; drop the
/// sentinel line. Return the rest (= the command's actual stdout).
/// Asymmetry: in PS-only mode, no echoed-command line exists — helper just
/// no-ops that step. In --ssh mode, ssh -tt echoes stdin → echo line is
/// present and gets stripped.
fn clean_stdout(buf: &str, uuid: &Uuid) -> String;
```

### `run_oneshot` flow (high-level)

Single function in `main.rs`. Owns the writer (`Arc<Mutex<Box<dyn Transport>>>`), the reader (`Box<dyn Transport>`), and a `stop: Arc<AtomicBool>` shared with `heartbeat_thread`. Uses a small state machine over the byte buffer:

```
[expect-host-prompt] ─(PS prompt seen)─►
   if --ssh: send "ssh -tt ALIAS\r"
             ─►[expect-remote-prompt]─(bash prompt seen)─► [send-cmd]
   else:    ─► [send-cmd]
[send-cmd] ─(send formatted command)─► [expect-sentinel]
[expect-sentinel] ─(sentinel line matched)─► [done] → clean_stdout, exit
[expect-sentinel] ─(timeout)─► exit 124
```

Reading is done synchronously from the reader Box right inside `run_oneshot` — no separate reader thread (we don't need to also pump stdin). Heartbeat thread shares the writer mutex, same as `bridge_loop`.
